# Biome Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace oxlint and prettier with Biome for both linting and formatting, adopting Biome's recommended defaults.

**Architecture:** Swap four packages (oxlint, @standard-config/oxlint, oxlint-tsgolint, prettier) for one (@biomejs/biome). Replace three config files (.prettierrc, .prettierignore, oxlint.config.ts) with one (biome.json). Update scripts and git hooks to use Biome CLI commands.

**Tech Stack:** Biome (latest), Bun, Lefthook

---

### Task 1: Remove old packages and install Biome

**Files:**
- Modify: `package.json`

- [ ] **Step 1: Remove old linting/formatting packages**

```bash
bun remove oxlint @standard-config/oxlint oxlint-tsgolint prettier
```

- [ ] **Step 2: Install Biome**

```bash
bun add --dev @biomejs/biome
```

- [ ] **Step 3: Verify installation**

```bash

```

### Task 2: Create biome.json and remove old config files

**Files:**
- Create: `biome.json`
- Delete: `.prettierrc`
- Delete: `.prettierignore`
- Delete: `oxlint.config.ts`

- [ ] **Step 1: Initialize Biome config**

```bash
bunx biome init
```

This creates a `biome.json` with the correct schema URL for the installed version.

- [ ] **Step 2: Update biome.json with project settings**

Replace the generated `biome.json` content with:

```json
{
	"$schema": "./node_modules/@biomejs/biome/configuration_schema.json",
	"organizeImports": {
		"enabled": true
	},
	"linter": {
		"enabled": true,
		"rules": {
			"recommended": true
		}
	},
	"formatter": {
		"enabled": true
	},
	"files": {
		"ignore": ["dist", ".output", "src/routeTree.gen.ts"]
	}
}
```

Notes:
- `organizeImports` sorts and groups imports automatically.
- Biome's formatter defaults: tabs, 80-char line width, LF line endings.
- `node_modules` is ignored by default, no need to list it.
- `src/routeTree.gen.ts` is auto-generated by TanStack Router, must be ignored.

- [ ] **Step 3: Delete old config files**

```bash
rm .prettierrc .prettierignore oxlint.config.ts
```

- [ ] **Step 4: Verify Biome loads the config**

```bash
bunx biome check --max-diagnostics=5 src/components/header.tsx
```

Expected: Biome runs and reports diagnostics (formatting diffs and/or lint warnings). No config errors.

### Task 4: Update lefthook.yml

**Files:**
- Modify: `lefthook.yml`

- [ ] **Step 1: Replace pre-commit jobs with Biome commands**

Replace the full contents of `lefthook.yml` with:

```yaml
pre-commit:
  parallel: true
  jobs:
    - name: biome-check
      glob: "*.{js,jsx,ts,tsx,css,json}"
      run: bunx biome check --fix {staged_files}
      stage_fixed: true

commit-msg:
  jobs:
    - name: commitlint
      run: bunx commitlint --edit {1}
```

Notes:
- The two old pre-commit jobs (lint-fix for JS/TS, format for CSS/JSON/MD/YAML) are consolidated into one `biome check --fix` job.
- Biome handles linting + formatting + import sorting in a single `biome check` command.
- Biome does not support YAML, Markdown, or HTML formatting. Those file types are dropped from the pre-commit hook. This is acceptable since prettier is being fully removed.
- The `commit-msg` job for commitlint is unchanged.

**Files:**
- Modify: all `.ts`, `.tsx`, `.css`, `.json` files in `src/`

```bash

```

This will reformat all files to Biome's style and apply safe lint fixes.

- [ ] **Step 2: Review the changes**


Verify the changes are formatting/lint-related only (whitespace, quotes, semicolons, import ordering, etc.). No logic changes should appear.

- [ ] **Step 3: Verify build still works**

```bash
bun run build
```

Expected: Build completes successfully with no errors.

- [ ] **Step 4: Verify lint passes clean**

```bash
bun run lint
```

Expected: Exit code 0, no diagnostics.

- [ ] **Step 5: Commit reformatted code**

```bash

```
